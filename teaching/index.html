Core Methodology

How GUS Works

Three measurements on your dataset, made before training any model, reveal whether a GNN will beat XGBoost.

1

Combine graph utility with improvement headroom. This is the single deciding metric.

Formula

Step 1 — Feature Predictiveness in Detail

Train the simplest possible model using only tabular features — no graph at all. If features already nail 95% AUC, the graph structure has nothing left to add. This step alone can save you weeks.

FP > 0.95
Features nearly perfect → Stop here. Use XGBoost.
FP 0.70 – 0.95
Features moderate → Proceed to Step 2.
FP < 0.70
Features weak → Graph may help. Proceed.

Step 2 — Fraud Homophily in Detail

Build a graph by connecting transactions that share an entity (same IP, same user, same device). Then measure: do fraudsters cluster with other fraudsters? Message passing only helps if they do — otherwise it averages the fraud signal away.

FH > 0.5
Fraud clusters together → GNN message passing amplifies signal.
FH < 0.5
Fraud is scattered → Message passing destroys signal.
# From analyze_datasets.py — compute_homophily() def compute_homophily(edge_index, y): src = edge_index[0] dst = edge_index[1] # Fraction of edges between same-label nodes same_label = (y[src] == y[dst]).float().mean() return same_label.item() # Real values: # IP Blocklist: 0.777 → Strong fraud clustering # Ecommerce: 0.734 → Good clustering # Credit Card: 0.041 → Fraud nearly isolated — GNN won't help

Step 3 — Effective Graph Utility in Detail

Raw Graph Utility (GNN AUC − XGB AUC) is misleading. If XGBoost already scores 0.9997, there's no room for improvement — EGU = near zero regardless. Effective Graph Utility corrects for this by scaling by improvement headroom.

EGU = Graph_Utility × (1 − XGB_AUC)
Where: Graph_Utility = Adjusted_Homophily × (1 − Feature_Predictiveness)
# From complete_gnn_experiments.py — DatasetSpec @property def effective_gu(self) -> float: return self.graph_utility * (1 - self.xgb_auc) # The key insight: # Customs: GU=0.11, XGB=0.9997 → EGU = 0.000033 → XGBoost wins # IP Bloc: GU=0.29, XGB=0.9116 → EGU = 0.0256 → GNN wins # Ecomm: GU=0.06, XGB=0.7808 → EGU = 0.0132 → GNN wins if self.effective_gu > 0.01: return "RECOMMEND: Build the GNN" else: return "RECOMMEND: Use XGBoost"

Evaluation

GUS Evaluation Metrics

Standard accuracy is useless in fraud detection — class imbalance makes it meaningless. GUS uses ranking-based and operational metrics.

📈
AUC-ROC
P(fraud score > legit score). Primary model selection metric. Threshold-independent.
0.5 = random · 1.0 = perfect · Used for all model comparisons
🎯
Average Precision
Area under the Precision-Recall curve. More sensitive than AUC at very low fraud rates.
Preferred when fraud rate < 1%. AP ≫ AUC at extreme imbalance.
🔍
Precision@K%
Of the top K% flagged transactions, how many are actually fraud?
K = 1%, 5%, 10%. Real operational constraint: analysts can only review K% per day.
🎣
Recall@K%
Of all true fraud, what fraction is caught in the top K%?
Measures detection coverage under budget constraint.
🚀
Lift@K%
P@K% ÷ base fraud rate. How much better than random inspection?
Lift=10 means top 1% has 10× more fraud than a random 1% would contain.
📊
NDCG
Normalized Discounted Cumulative Gain. Rewards finding fraud at the very top of the ranking list.
More credit for catching fraud at rank 1 than at rank 100.
